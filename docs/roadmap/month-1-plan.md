# Month-One Build Plan

## Guiding Constraint

Build fast enough to ship a credible MVP within one month, but do not compress the product into a one-day tutorial app. Each phase should leave behind working code, tests where risk justifies them, and updated documentation.

## Phase 1: Foundation

Target: Days 1-5

- monorepo setup
- app scaffolds
- local Mongo/Redis
- auth approach
- initial schemas
- layout system
- documentation system

## Phase 2: Reading Identity

Target: Days 6-10

- signup/login
- reading profile
- favorite genres/authors
- liked/disliked/completed/DNF books
- available time and speed estimation

## Phase 3: Book Catalog and Events

Target: Days 11-15

- book and author CRUD
- admin book management
- reading event tracking
- DNF records
- catalog search/filtering

## Phase 4: Decision Engine MVP

Target: Days 16-22

- candidate generation
- weighted scoring
- anti-DNF penalty
- mood/time fit
- explanation generator
- recommendation feedback loop

## Phase 5: Admin Analytics and Polish

Target: Days 23-27

- recommendation tuning
- drop-off analytics
- popular recommendation paths
- dashboard metrics
- frontend states and polish

## Phase 6: Deployment and Demo Readiness

Target: Days 28-30

- production env setup
- deployment docs
- seed data
- demo script
- recruiter-facing README
- interview talking points
